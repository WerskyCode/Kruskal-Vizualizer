## 1. Введение
### 1.1. Назначение программы
Программное обеспечение предназначено для наглядной демонстрации работы жадного алгоритма Краскала при построении МОД во взвешенном неориентированном графе.

### 1.2. Область применения
Инструмент ориентирован на использование в образовательных целях студентами и преподавателями в рамках дисциплин «Алгоритмы и структуры данных» для глубокого понимания работы Системы Непересекающихся Множеств (СНМ) и теории графов.

### 1.3. Целевая аудитория
1) Студенты, изучающие алгоритмы и структуры данных
2) Преподаватели для демонстрации работы алгоритма Краскала
3) Разработчики, изучающие Kotlin и визуализацию алгоритмов

### 1.4. Термины и определения

| Термин | Определение |
|--------|-------------|
| Граф | Совокупность вершин и соединяющих их ребер |
| Вершина | Базовый элемент, точка пересечения или объект, между которыми устанавливаются связи (рёбра) |
| Ребро | Связь между двумя вершинами, имеющая вес |
| Вес ребра | Числовая характеристика ребра (стоимость, расстояние) |
| Минимальное остовное дерево (МОД) | Остовное дерево графа, имеющее минимальный возможный вес, где под весом дерева понимается сумма весов входящих в него рёбер |
| Алгоритм Краскала | Жадный алгоритм построения MST, сортирующий ребра по весу и добавляющий их, если они не образуют цикл |
| СНМ (DSU) | Система непересекающихся множеств — структура данных для проверки принадлежности элементов к одному множеству |
| Визуализация | Графическое представление работы алгоритма с подсветкой шагов |
| Пошаговый режим | Выполнение алгоритма по одному шагу с задержкой или по кнопке |
| Интерактивность | Возможность взаимодействия пользователя с графом (добавление(/удаление) вершин(/ребер) |

## 3. Ожидаемые функциональные возможности
### 3.1. Модуль интерактивного ввода и управления графом
#### 3.1.1. Редактор холста
Редактор холста предоставляет пользователю интуитивно понятный способ создания графа без необходимости писать код или работать с файлами. Весь процесс построения графа выполняется с помощью мыши и клавиатуры.

**Интерактивное добавление вершин** 
Клик левой кнопкой мыши  по свободному месту на холсте. В точке клика появляется вершина с уникальным номером (ID). Каждая новая вершина получает следующий по порядку номер (0, 1, 2, …).

Визуальное представление:
1) Вершина отображается как закрашенный круг.
2) Внутри круга её номер.
3) При наведении курсора на вершину она должна подсвечиваться 

**Проведение взвешенных ребер**
Пользователь может последовательно кликать по двум вершинам , выделяя их, и между ними образуется ребро. После этого появляется поле для ввода веса ребра (целое число).

Визуальное представление:
1) Ребро отображается как линия между вершинами
2) над ребром отображается его вес
3) Возможность редактирования веса после создания

Также альтернативно реализация drag-and-drope - возможность пользователю соединить вершины путем перетаскивания вершины 1 в вершину 2 

#### 3.1.2. Импорт файлов
1) Возможность загрузки структуры графа из текстового файла конфигурации
2) Поддержка формата: '[ID вершины1] [ID вершины2] [Вес]'

#### 3.1.3. Управление состоянием
1) Кнопка полной очистки холста для построения новой конфигурации графа
2) Сохранение текущего графа в файл

### 3.2. Модуль пошаговой визуализации и симуляции
#### 3.2.1. Дискретное управление
1) Шаг вперед — выполнение одной итерации алгоритма
2) Шаг назад — откат к предыдущему состоянию

#### 3.2.2. Автоматическая симуляция
1) Режим непрерывного воспроизведения (Auto-play)
2) Возможность постановки на паузу

#### 3.2.3. Анимационная индикация(Предварительно)
Цветовое кодирование ребер:
| Цвет | Значение |
|------|----------|
| Зеленый | Ребро успешно добавлено в каркас МОД |
| Желтый | Текущее рассматриваемое ребро на данном шаге итерации |
| Красный | Ребро не добавлено в каркас МОД (создает цикл) |
| Серый | Ребра, ожидающие очереди проверки |

#### 3.2.4. Работа программы (Алгоритм Краскала)
**Идея:**
Алгоритм Краскала ищет самый дешевый способ соединить все вершины так, чтобы между ними не образовались циклы. Для этого программа делает следующие шаги:

**Шаг A. Сортировка**
Как только пользователь нажимает кнопку запуска, программа сортирует все ребра графа по возрастанию веса. 

**Шаг B. Основной цикл**
Программа берет из начала очереди первое (самое дешевое) ребро, подсвечивает его желтым цветом(?) на экране и проверяет:
Случай 1 Цикла нет): Если эти две вершины еще ни разу не были соединены друг с другом (даже обходными путями через другие точки), алгоритм одобряет это ребро. Оно красится в зеленый цвет(?), добавляется в итоговое дерево, а в панель логов пишется оповещение об успешном добавлении ребра.
Сценарий 2 (Образуется цикл): Если к этим двум точкам уже можно добраться по другим зеленым линиям, то проводить между ними еще одну прямую линию нельзя, т.к получится цикл. Алгоритм отклоняет это ребро,  оно красится в красный цвет(?), а в лог пишется оповещение о невозможности соединения ребер.
**Шаг C.**
Программа повторяет Шаг B для каждого следующего ребра в очереди до тех пор, пока все точки на холсте не окажутся связаны в единую сеть. Как только это произойдет, алгоритм завершит работу и на холсте останется МОД.

**Сложность:** O(nlog(n))

### 3.3. Информационный логгер
#### 3.3.1. Текстовое сопровождение
Динамический вывод пояснений к каждому шагу алгоритма в специализированную текстовую панель для детального анализа работы СНМ на каждом шаге.

#### 3.3.2. Информационная статистика
1) Текущий вес MST
2) Количество добавленных ребер
3) Номер текущего шага / общее количество шагов

## 4. Формат входных конфигурационных файлов 
### 4.1. Структура файла
Входной файл импорта должен соответствовать строгому шаблону:
**Строка 1:** '[Количество вершин V] [Количество ребер E]'
**Строки от 2 до E+1:** '[ID начальной вершины] [ID конечной вершины] [Вес ребра (целое число)]'

### 4.2. Пример валидного файла конфигурации

```text
4 4
1 2 5
2 3 3
```

### 6.1. Надежность
1) Приложение корректно обрабатывает ошибки ввода: неверный формат файла, пустой граф, несвязный граф.

3) Некорректные действия пользователя блокируются (например, запуск алгоритма без графа).

### 6.2. Кроссплатформенность
1) Поддерживаемые ОС: Windows 10/11, Linux (Ubuntu 20.04+, Debian 11+), macOS (12+).
2) Для запуска требуется JVM версии 21 и установленное графическое окружение.
